# M20 literature review: improving the CGHSTC-34 activation equation

Date: 2026-06-01
Project: CGHSTC-34 / ApJ evolutionary-governor branch
Status: literature-guided design review; no new equation accepted yet

## Executive result

The present rho sequence is weak-to-moderate but directional:

- M9 rho: `0.319`
- M18 full-fit rho: `0.351`
- M18 out-of-fold rho: `0.289`
- M19 full-fit rho: `0.364`
- M19 out-of-fold rho: `0.355`

The literature and the data inspection point to the same issue: direct MSE/rank fitting against clipped `A_inferred` is probably the wrong statistical target. In the current M9 artifact, `A_inferred` has:

- `12` exact zeros;
- `57` exact ones;
- only `18` interior values.

That means `79%` of the 87-galaxy target vector is endpoint-clipped. A pointwise scalar equation has little room to rank objects inside the endpoint masses. A rho target above `0.60` is unlikely unless the next model treats the endpoints as censored or inflation masses, or moves to the velocity/BTFR domain where the observable is less degenerate.

The most promising new variable is not raw specific angular momentum `j` by itself. Direct proxies such as `j_obs ~ 2 R_d V_flat` correlate weakly with `A_inferred`. The best local angular-momentum-like proxy found in existing artifacts is a dimensionless `lambda_like_j_over_sqrtMR` proxy with Spearman rho about `0.353` against `A_inferred`, comparable to the full M19 model. This suggests that an angular-momentum state variable may matter, but probably as a normalized spin/settling variable or interaction term, not as raw `j`.

No direct observed `j_*`, `j_gas`, `j_bar`, `lambda`, or halo spin column was found. Therefore M20 should distinguish:

1. artifact-backed proxies that can be tested immediately;

## Mathematical/statistical literature lessons

### 1. Bounded/censored responses

Relevant anchors:

- Ferrari & Cribari-Neto, beta regression for rates/proportions.
- Ospina & Ferrari, zero-or-one inflated beta models.
- Tobin, Tobit/censored regression for limited dependent variables.

Lesson for CGHSTC-34:

The current target is a clipped activation estimate. Because most galaxies are at exactly 0 or 1, a beta-like or censored latent model is more appropriate than direct MSE on `A_inferred`. The equation should predict a latent `z_i`, with observed `A_inferred` generated by censoring:

```text
z_i = eta(g_i, J_i) + epsilon_i
A_i = 0      if z_i <= 0
A_i = z_i    if 0 < z_i < 1
A_i = 1      if z_i >= 1
```

Alternative zero/one-inflated form:

```text
P(A=0) = pi_0(g,J)
P(A=1) = pi_1(g,J)
A | 0<A<1 ~ Beta(mu(g,J), phi)
```

This should be the first serious attempt to reach rho over `0.60`, because it restores latent ordering within endpoint-clipped objects.

### 2. Rank-oriented learning

Relevant anchors:

- RankNet / LambdaRank / LambdaMART learning-to-rank literature.
- Pairwise ranking losses and concordance-index optimization.
- Isotonic regression and monotone calibration.

Lesson for CGHSTC-34:

If the objective is rank correlation, the loss must include rank. Instead of only minimizing squared error, use a pairwise logistic ranking component:

```text
L_rank = sum_{i<k, A_i != A_k} log(1 + exp[-s_ik (z_i - z_k)])
s_ik = sign(A_i - A_k)
```

The score can be combined with likelihood:

```text
L_total = L_censored_or_beta + alpha L_rank + complexity_penalty
```

This gives the model a direct route to improving rho while retaining physical likelihood constraints.

### 3. Sparse/nonlinear differential-equation discovery

Relevant retrieved/anchor works:

- Brunton, Proctor & Kutz: SINDy / sparse identification of nonlinear dynamics.
- Neural ODE / latent ODE literature.
- Rackauckas et al.: universal differential equations for scientific machine learning.

Lesson for CGHSTC-34:

Do not add an unconstrained neural ODE for 87 galaxies. Use a sparse, predeclared ODE library:

```text
dA/dtau = c1 D(1-A) + c2 D A(1-A) - c3 R A
        + c4 J A(1-A) + c5 D J (1-A) - c6 R J A
```

where `J` is a normalized angular-momentum/spin proxy. Constrain selected terms to 3--4 nonzero coefficients and require fold stability.

### 4. Galaxy angular momentum literature

Relevant anchors:

- Fall & Efstathiou; Fall: angular momentum in disk formation.
- Mo, Mao & White: disk formation in CDM; spin parameter controls disk scale and rotational support.
- Romanowsky & Fall; Fall & Romanowsky: stellar specific angular momentum, mass, and morphology relation.
- Obreschkow & Glazebrook: baryonic/angular-momentum relations and disk stability.
- SPARC/RAR/BTFR literature from Lelli, McGaugh, Schombert and collaborators.

Lesson for CGHSTC-34:

Angular momentum should be tested as a state variable describing whether the forming galaxy is rotationally organized enough to sustain the BDVR response. It should probably enter as one of:

1. drive term: high settled spin increases activation onset;
2. detuning term: high extended H I angular momentum keeps the object in a diffuse reservoir state;
3. memory/initial condition: angular momentum sets `A0` or relaxation time;
4. velocity-domain structural term: angular momentum affects disk scale and therefore the BDVR response through `R_d`, `R_HI`, and `Vflat`.

Raw `j` is not enough. The better candidate is a normalized spin/compactness state:

```text
J_lambda = log[(2 R_d Vflat) / sqrt(M_b R_d)]
```

or a baryonic partitioned version:

```text
J_bar = f_* (2 R_d Vflat) + f_gas (R_HI Vflat)
J_settle = J_star / (J_star + J_HI)
```

Only the first can be tested immediately with the current artifacts. The partitioned form requires more defensible gas and stellar angular-momentum assumptions.

## Equation families recommended for M20

### M20A: censored latent activation with angular momentum

```text
z_i = beta0 + beta_D D_i - beta_R R_i + beta_J J_i + beta_DJ D_i J_i - beta_RJ R_i J_i + epsilon_i
A_obs = clip(z_i, 0, 1)
```

Selection objective: censored likelihood + rank loss.

Why it may improve rho: endpoint-clipped galaxies recover latent ordering through `z_i`.

### M20B: zero/one-inflated beta activation

```text
P(A=0) = logistic(h0(g,J))
P(A=1) = logistic(h1(g,J))
mu = logistic(hmu(g,J))
A_interior ~ Beta(mu phi, (1-mu) phi)
```

Why it may improve rho: endpoint class and interior activation are different problems.

### M20C: pairwise-rank ODE

```text
dA/dtau = F(A,D,R,J)
L = MSE_or_likelihood + alpha L_rank
```

Why it may improve rho: direct rank-pressure while preserving bounded dynamics.

### M20D: sparse SINDy-style ODE

Candidate term library:

```text
D(1-A), D A(1-A), -R A, J(1-A), J A(1-A), D J(1-A), -R J A
```

Why it may improve rho: finds only the angular-momentum interaction terms that matter.

### M20E: velocity-domain latent BDVR likelihood

```text
A_latent = state_equation(g,J)
Q_eff = Q_base + A_latent * Q_response(g,J)
log Vflat ~ Normal(log V_BDVR(Q_eff), sigma)
```

Why it may improve rho: avoids using a clipped derived activation target as the primary observable.

## Recommended order of execution

1. M20A censored latent activation with `J_lambda`.
2. M20C pairwise-rank augmented version of M20A.
3. M20D sparse ODE with angular-momentum interaction terms.
4. M20E velocity-domain likelihood if M20A/C/D cannot push out-of-fold rho above `0.60`.
5. External angular-momentum ingestion only if local proxies plateau below the target.

## Acceptance gates

Do not accept a model solely because full-fit rho exceeds `0.60`. Require:

- five-fold out-of-fold rho > `0.60`;
- endpoint handling documented and tested;
- prediction distribution not saturated;
- no per-galaxy tuning;
- fold-selected terms stable in at least 4 of 5 folds;
- angular-momentum term improves out-of-fold rho by at least `0.05` over the same model without it;
- permutation test or bootstrap confidence interval indicates the improvement is not a small-N accident.

## Bottom line

The literature-guided path is not another hand-built gate. The next model should change the statistical object: fit latent/censored activation or velocity likelihood, add rank-aware loss, and test angular momentum as a normalized state variable. The existing local data support an immediate proxy test using `lambda_like_j_over_sqrtMR`, but direct `j` alone is unlikely to solve the correlation problem.
